Integrating into Java applications

Download Java client 1.0.3

See com.yoursway.crashkit.demo.Main.java in com.yoursway.crashkit.demo for usage. (Note that com.yoursway.crashkit can be used both in plain Java and in OSGi environment.)

Integrating into PHP applications

Download PHP client 1.0.7 (includes crashkit.php and a demo application)

Step 1. Drop “crashkit.php” into your project.

Step 2. Copy and paste the following text into the beginning of your main PHP file(s):

<?php
  define('CRASHKIT_PRODUCT', '{{account.permalink|escape}}/{{product.unique_name|escape}}');
  define('CRASHKIT_DIE_MESSAGE',
    '<title>Server error</title>' .
    '<div style="width: 600px; margin: 50px auto 0px auto; font: 14px Verdana, sans-serif;">' .
    '<b style="color: red;">Server error</b>' .
    '<p>Sorry, we have failed to process your request now. Please try again later.' .
    '<p>Our developers have just been notified about this error.' .
    '<p>If you need help immediately, please e-mail <a href="mailto:{{user.email}}">{{user.email}}</a>.' .
    '</div>');
  require_once 'crashkit.php';
?>

You can customize the error message if you don't like the default one.

Integrating into HTML/JavaScript code

Just drop the following SCRIPT tag into your HTML page:

<script src="http://crashkitapp.appspot.com/static/javascript/crashkit-js-api.js?{{account.permalink|escape}}/{{product.unique_name|escape}}" type="text/javascript" charset="utf-8"></script>

After this, IE and Firefox will report all occurred exceptions automatically. Unfortunately, Safari and Google Chrome won't.

To also support Safari and Chrome, you need to manually catch all exceptions and feed them into CrashKit using CrashKit.report(exception) method: 

try { ...code here.. } catch(e) { CrashKit.report(e); }

If you are using jQuery, CrashKit automatically wraps all your event handlers into try-catch blocks.
